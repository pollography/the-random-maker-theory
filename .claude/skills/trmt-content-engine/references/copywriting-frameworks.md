# Copywriting Frameworks — TRMT Referenz

Jeder Artikel braucht einen Hook und eine Struktur. Hier sind die Frameworks, angepasst an den TRMT-Stil.

## 1. PAS — Problem, Agitate, Solution

**Wann:** Tutorials, How-Tos, Tool-Vergleiche

**Schema:**
1. **Problem:** Benenne das Problem konkret
2. **Agitate:** Zeig warum es nervt / was es kostet
3. **Solution:** Deine Lösung/Empfehlung

**TRMT-Beispiel:**
```
# n8n Workflows automatisch backuppen

Du hast 20 Workflows in n8n gebaut. Alles läuft. Dann machst du ein
Update und — weg. Alles weg. Kein Backup, kein Zurück.

Mir ist das passiert. Zwei Wochen Arbeit. Ich war echt angepisst.

Seit dem hab ich ein automatisches Backup-System. Dauert 10 Minuten
zum Einrichten. Zeig ich dir hier.
```

## 2. BAB — Before, After, Bridge

**Wann:** Reviews, Transformations-Stories, Vorher/Nachher

**Schema:**
1. **Before:** So war es vorher (Pain)
2. **After:** So ist es jetzt (Gain)
3. **Bridge:** Das Tool/die Methode die den Unterschied macht

**TRMT-Beispiel:**
```
# Von Chaos zu System: Mein Obsidian Setup

Vorher: 200 Notizen in Apple Notes. Null Struktur. Ich hab nie was
wiedergefunden.

Jetzt: Alles in Obsidian. Verlinkt. Durchsuchbar. Ich find in 5
Sekunden was ich letzte Woche über ESP32-Sensoren notiert hab.

Der Umstieg hat einen Nachmittag gedauert. So hab ich's gemacht.
```

## 3. AIDA — Attention, Interest, Desire, Action

**Wann:** Product Reviews, Kaufberatungen, Newsletter

**Schema:**
1. **Attention:** Aufmerksamkeit mit starkem Opening
2. **Interest:** Details die neugierig machen
3. **Desire:** Konkreter Nutzen / Warum du das willst
4. **Action:** Was du jetzt tun sollst

**TRMT-Beispiel:**
```
# Claude vs ChatGPT für Coding: Mein ehrlicher Vergleich

[A] Ich hab beide 3 Monate lang für echte Projekte genutzt. Täglich.

[I] Das Ergebnis war... nicht das was ich erwartet hab. In manchen
Bereichen gewinnt ChatGPT klar. In anderen ist Claude meilenweit
voraus.

[D] Am Ende hab ich für mich eine klare Aufteilung gefunden die
mir pro Woche ca. 5 Stunden spart.

[A] Hier ist mein Setup.
```

## 4. StoryBrand (vereinfacht)

**Wann:** Maker-Projekte, Life-Learning, persönliche Posts

**Schema:**
1. **Held:** Der Leser (oder TRMT) hat ein Problem
2. **Guide:** TRMT zeigt den Weg (Empathie + Autorität)
3. **Plan:** Konkrete Schritte
4. **Ergebnis:** Was passiert wenn man's macht / nicht macht

**TRMT-Beispiel:**
```
# Mein erstes ESP32-Projekt: Temperatur-Sensor fürs Homelab

Du willst wissen ob dein Server-Rack zu warm wird. Klar, du könntest
ein fertiges Monitoring kaufen. Oder du baust dir für 8 Euro selbst was.

Ich hab das in einem Abend hinbekommen. Ohne Löterfahrung. Hier ist
der komplette Weg von "ESP32 was?" bis "Dashboard zeigt 23.4°C".
```

## 5. R-C-C-O-V — Research, Claim, Counter, Opinion, Verdict

**Wann:** Deep-Dives, Analysen, kontroverse Themen

**Schema:**
1. **Research:** Was sagen die Daten/Quellen?
2. **Claim:** Die gängige Meinung
3. **Counter:** Gegenargumente / was dagegen spricht
4. **Opinion:** TRMTs persoenliche Einschaetzung
5. **Verdict:** Klares Fazit

**TRMT-Beispiel:**
```
# Sind Open-Source LLMs wirklich besser? Meine Analyse

[R] Laut den letzten Benchmarks schlägt Llama 3 viele proprietäre
Modelle. Die Community feiert.

[C] "Open Source wird alles dominieren" ist grad der Konsens.

[CO] Aber: Fine-Tuning kostet. Hosting kostet. Und die wirklich
guten Modelle kommen immer noch von Anthropic und OpenAI.

[O] Ich find die Wahrheit liegt — wie so oft — in der Mitte.
Für bestimmte Use-Cases sind Open-Source Modelle perfekt.
Für andere verschwendest du nur Zeit.

[V] Mein Fazit: Nutz beides. Hier ist wann ich was nehme.
```

## Framework-Auswahl nach Content-Typ

| Content-Typ | Empfohlenes Framework |
|---|---|
| Tutorial / How-To | PAS |
| Tool-Review | BAB oder AIDA |
| Vergleich | R-C-C-O-V |
| News-Roundup | Direct (kein Framework nötig) |
| Maker-Projekt | StoryBrand |
| Life-Learning | StoryBrand oder BAB |
| Deep-Dive / Analyse | R-C-C-O-V |
| Kaufberatung | AIDA |

## Hook-Regeln (Framework-übergreifend)

1. **Erste 2 Sätze entscheiden ob jemand weiterliest**
2. Kein "In diesem Artikel werde ich..." — langweilig
3. Starte mit: Konkretem Fakt, Story, Problem, oder kontroversem Take
4. Vermeide Fragen als Hook (wirkt billig: "Hast du dich schon mal gefragt...?")
5. Maximal 3 Sätze bis der Leser weiß worum es geht
